# Vibe Coding: Verify What AI-Generated Code Actually Does

You asked Claude, Cursor, or Copilot to build something. It generated hundreds of lines. It looks right. But does it actually work the way you expect? What types flow through those functions? What happens when edge cases hit? What if there's a subtle bug that only shows up at runtime?

**41% of all code is now AI-generated. Amazon Q's AI code just caused 6.3M lost orders.** The gap between "code that compiles" and "code that works correctly" is where bugs hide — and trickle closes that gap.

## The Problem

AI-generated code has three failure modes:

1. **Type mismatches** — The AI returns `string` where your code expects `number`. TypeScript catches some of this, but not runtime behavior.
2. **Silent wrong behavior** — The function runs without errors but returns the wrong result. No test catches it because no tests exist yet.
3. **Missing edge cases** — The AI handles the happy path but crashes on null, empty arrays, or unexpected input shapes.

You can't review 500 lines of AI code by reading it. You need to run it and see what it actually does.

## The Solution: Observe, Understand, Verify

```bash
npm install -g trickle-cli
```

### Step 1: Run the AI-generated code

```bash
trickle run node app.js
```

Zero code changes needed. Trickle captures every function call, every variable assignment, every API response, every error — automatically. Works with Express, Fastify, Koa, Hono, and plain Node.js.

### Step 2: Understand what it does

```bash
trickle explain src/app.js
```

See every function's actual input/output types, call graph, and data flow — based on real execution, not static analysis. If the AI created a function that takes `{user: {name: string, email: string}}` but your database returns `{user_name: string, user_email: string}`, you'll see it immediately.

```bash
trickle summary
```

Get a complete overview: how many functions, what errors occurred, any N+1 queries, slow functions, root causes.

### Step 3: Generate a test suite from what you observed

```bash
trickle test --generate --unit -o tests/generated.test.ts
```

Trickle generates real unit tests using the actual inputs and outputs it observed at runtime. Not mocked data — real data from real execution.

```typescript
// Auto-generated by trickle from runtime observations
import { describe, it, expect } from "vitest";
import { calculateTotal, formatUser } from "./src/utils";

describe("calculateTotal", () => {
  it("given [29.99, 49.99], returns 79.98", () => {
    const result = calculateTotal([29.99, 49.99]);
    expect(typeof result).toBe("number");
  });
});

describe("formatUser", () => {
  it("given {first, last, email}, returns {fullName, email, active}", () => {
    const result = formatUser({ first: "Alice", last: "Smith", email: "alice@test.com" });
    expect(typeof result.fullName).toBe("string");
    expect(typeof result.email).toBe("string");
    expect(typeof result.active).toBe("boolean");
  });
});
```

For Python:
```bash
trickle test --generate --unit --framework pytest -o tests/test_generated.py
```

### Step 4: Generate typed API clients

```bash
trickle codegen --client -o src/api-client.ts
```

If the AI generated a REST API, trickle creates a fully typed client with every endpoint, request body, and response shape — derived from actual runtime behavior.

### Step 5: Check for security issues

```bash
trickle security
```

Scan for prompt injection, data exfiltration, sensitive data in API responses, and other vulnerabilities that AI-generated code commonly introduces.

### Step 6: Set a baseline and guard against regressions

```bash
trickle check --save baseline.json
```

Commit `baseline.json`. On every future change (AI-generated or not):

```bash
trickle run node app.js
trickle check --against baseline.json
```

This catches breaking changes: removed fields, type changes, missing endpoints.

## The Full Workflow

```
AI generates code
       ↓
trickle run node app.js        ← observe everything
       ↓
trickle explain src/app.js     ← understand what it does
trickle summary                ← errors, queries, root causes
       ↓
trickle test --generate --unit ← generate test suite
trickle codegen --client       ← generate typed API client
trickle security               ← scan for vulnerabilities
       ↓
trickle check --save baseline  ← lock in current behavior
       ↓
Ship with confidence
```

## Why This Matters

| Without trickle | With trickle |
|---|---|
| Read 500 lines of AI code and hope | Run it once and see exactly what it does |
| Write tests manually for code you didn't write | Auto-generate tests from observed behavior |
| Hope the types are right | See actual runtime types |
| Ship and find bugs in production | Catch mismatches before deploy |
| No documentation for the AI's code | Auto-generated types, API specs, test suite |

## Works With Every AI Coding Tool

- **Claude Code**: `trickle init` sets up MCP tools so Claude can debug autonomously
- **Cursor**: Use `trickle run` before and after AI edits to verify changes
- **GitHub Copilot**: Generate code, then `trickle run` + `trickle test --generate`
- **ChatGPT / GPT-4**: Paste generated code, run with trickle, verify

## Real Example: AI Generates an Express API

```
You: "Create a REST API for a blog with posts and comments"
AI: generates app.js with Express routes
```

```bash
# 1. Run it
trickle run node app.js

# 2. Hit the endpoints
curl http://localhost:3000/api/posts
curl -X POST http://localhost:3000/api/posts -d '{"title":"Test","body":"Hello"}'

# 3. See what trickle captured
trickle functions
# → GET /api/posts, POST /api/posts, GET /api/posts/:id, ...

trickle codegen
# → TypeScript interfaces for every endpoint

trickle test --generate -o tests/api.test.ts
# → Ready-to-run vitest tests for every route

trickle test --generate --unit -o tests/unit.test.ts
# → Unit tests for every observed function
```

In 60 seconds, you went from "AI generated some code" to "I have types, tests, and API docs for everything it built."
